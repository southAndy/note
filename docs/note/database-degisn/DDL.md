#database 

DDL 英文全名為 Data Definition Language，可以翻譯為 資料定義語言，用途是就是把「設計好的資料庫結構」轉化成可執行、可版本控、可自動化的程式碼。


## 好處

1. 文件化定義資料表
2. 可以結合文件生成工具產生
3. 可以版本控制、追溯
4. 可以被自動化執行


## 基本架構

以下是一張表的架構圖

```sq
'Action' Table 'TableName' (
	'欄位名稱' '資料型別' '約束條件' '預設值',
)
```

`Action`：**這邊定義對表的操作**，可以是新增(CREATE),編輯(ALTER),刪除(DROP)。
`TableName`：就是定義表的名稱
`key`：就是這張表要有的欄位

> 不同 DB 語言所產生的架構都不同，這邊是以 MySQL 為例
