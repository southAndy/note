#obsidian #thinking 

 1. **一張筆記一個概念**：例如現在有一份筆記，裡面包含了 vue 的響應式、 vue 元件的生命週期，就應該至少拆成兩份筆記(響應式、生命週期)
	 1. **好處**：提高複用性，因為如果文章內包含不只一個概念，在重複使用時就比較會遇到不必要的概念也被引入。
	 2. **壞處**：**要花更多腦袋去思考、拆分每個筆記**，建議要在精神好時慢慢處理
2. 目前架構：
	1. inbox 紀錄即時的發想
	2. literature 紀錄有興趣的文獻、連結（**最好能加上自己為何好奇？**）
	3. permanent 永久筆記（**我現在會理解為想要永久留下的知識整理**）

3. 加入 **延伸閱讀**  的連結是這個概念的延伸，例如在 computed 的文章內，延伸閱讀可能就會有：
	1. computed vs method 兩者的差異
	2. computed 的使用限制
